# 谷歌广告原生实现方案

## 1. 目标

当前项目已经移除 `@nuxtjs/google-adsense` 模块，后续统一使用原生 Google AdSense 接入方式。

本次实现目标：

- 使用原生脚本加载 AdSense
- 在首页文章列表中间插入一个展示广告位
- 保持广告卡片与文章卡片同级布局
- 开发环境不强行请求真实广告，避免本地调试干扰

## 2. 当前落地方式

### 2.1 全局脚本注入

在 [nuxt.config.ts](/Users/wangpenglong/projects/nuxt/Ideaflow-web-blog/nuxt.config.ts) 的 `app.head.script` 中注入：

```html
<script async src="https://pagead2.googlesyndication.com/pagead/js/adsbygoogle.js?client=ca-pub-1870872898412136" crossorigin="anonymous"></script>
```

这样做的好处是：

- 不依赖 Nuxt 模块
- 所有页面只加载一次脚本
- 组件内部只负责渲染广告位和执行 `push`

### 2.2 首页中部广告卡片

广告卡片组件：

- [GoogleAdsenseCard.vue](/Users/wangpenglong/projects/nuxt/Ideaflow-web-blog/app/components/Ad/GoogleAdsenseCard.vue)

首页文章列表入口：

- [AsmrAd.vue](/Users/wangpenglong/projects/nuxt/Ideaflow-web-blog/app/components/Blog/ArticleLayout/Card/AsmrAd.vue)

当前策略：

- 基于“当前分页结果”的文章数量计算中点
- 将广告卡片插入到中间位置
- 当分页大小为 `9` 时，广告通常会落在第 `5` 个卡片位置附近

## 3. 配置项

当前通过 `runtimeConfig.public` 暴露两个广告参数：

- `googleAdsenseClient`
- `googleAdsenseHomeSlot`
- `enableDevAdsense`

定义位置：

- [nuxt.config.ts](/Users/wangpenglong/projects/nuxt/Ideaflow-web-blog/nuxt.config.ts)

默认值：

- `googleAdsenseClient`: `ca-pub-1870872898412136`
- `googleAdsenseHomeSlot`: `9250379418`
- `enableDevAdsense`: `false`

如果后续希望按环境切换，建议在环境变量中覆盖：

```bash
NUXT_PUBLIC_GOOGLE_ADSENSE_CLIENT=ca-pub-1870872898412136
NUXT_PUBLIC_GOOGLE_ADSENSE_HOME_SLOT=9250379418
NUXT_PUBLIC_ENABLE_DEV_ADSENSE=true
```

## 4. 开发环境说明

开发环境下当前策略是：

- 仍然加载页面结构
- 但广告组件默认不请求真实 Google 广告
- 页面展示一个占位广告卡片，方便检查插入位置、尺寸和列表节奏
- 如需在开发环境主动测试真实广告，可显式设置 `NUXT_PUBLIC_ENABLE_DEV_ADSENSE=true`

这样处理的原因：

- 本地 `localhost` 或开发域名下，真实广告通常不会稳定展示
- 浏览器广告拦截插件也会影响调试结果
- 在开发环境直接请求广告，容易产生误判，影响页面调试体验

所以结论是：

- 开发环境可以直接看到“广告位置”
- 默认不会直接看到真实广告内容
- 只有显式打开 `NUXT_PUBLIC_ENABLE_DEV_ADSENSE=true` 后，开发环境才会尝试请求真实广告
- 真实广告展示请以 `preview`、测试域名或正式域名为准

## 5. 生产环境渲染逻辑

生产环境下广告组件会：

1. 挂载后渲染 `<ins class="adsbygoogle">`
2. 执行：

```js
(window.adsbygoogle = window.adsbygoogle || []).push({})
```

这样即可按 Google AdSense 官方 HTML 方案完成广告请求。
